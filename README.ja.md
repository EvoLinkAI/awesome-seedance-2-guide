## ✨ なぜ Seedance 2.0 なのか？

**画像 + ビデオ + オーディオ + テキスト**の4つのモダリティを同時入力として対応しています。`@素材名`を自然言語で使用して、希望する効果を説明すると、モデルが理解します。単なる「生成」ではなく、真の制御可能な創作です。

## 📋 パラメータ仕様

| 入力タイプ | サポートされている形式 | 数量制限 | サイズ制限 | 期間制限 |
|----------|----------|----------|----------|----------|
| 画像 | jpeg、png、webp、bmp、tiff、gif | ≤ 9 | 単一 < 30MB | — |
| ビデオ | mp4、mov | ≤ 3 | 単一 < 50MB | 合計 2-15s |
| オーディオ | mp3、wav | ≤ 3 | 単一 < 15MB | 合計 ≤ 15s |
| テキスト | 自然言語 | — | — | — |

**組み合わせ制限：** 合計 ≤ 12 ファイル（画像 + ビデオ + オーディオ合計）

**出力仕様：** 生成期間 4-15s 自由に選択可能、ビデオ最大 720p、サウンドエフェクト/背景音楽を含む

> **コンプライアンスに関する注記：** 現在、リアルな人間の顔を含む素材のアップロードはサポートされていません。イラストレーションスタイル、AI生成の仮想キャラクター、動物、製品、シーンなどの使用をお勧めします。

## 🎮 インタラクション方法

プロンプトで `@素材名` を使用して、各素材の役割を指定します。アップロード順序は番号付け順序です：

```
@画像1 を最初のフレームとして、@ビデオ1 をカメラ言語の参照として、@オーディオ1 を背景音楽として
```

| エントリーポイント | ユースケース |
|------|----------|
| **最初/最後のフレーム** | 最初のフレームのみ（または最初 + 最後のフレーム）+ プロンプトをアップロード |
| **オールインワン参照** | マルチモーダル組み合わせ入力（画像 + ビデオ + オーディオ + テキスト） |

**一般的な書き方パターン：**

```
# 最初のフレームを指定
@画像1 をシーンの最初のフレームとして使用、...

# カメラ移動のみを参照、キャラクターは参照しない
@ビデオ1 のすべてのカメラ移動効果を参照、ただしキャラクターの外観は @画像1 を使用

# アクションとカメラ移動を分けて参照
@ビデオ1 からキャラクターアクションを参照、@ビデオ2 から円形カメラ移動を参照

# ビデオ延長（生成期間 = 新しい秒数、合計期間ではない）
@ビデオ1 を 5 秒延長、[コンテンツ説明]

# ビデオサウンドエフェクトを参照
背景BGMは @ビデオ1 のサウンドエフェクトを参照
```

## 💡 高度なテクニック

**プロンプト作成**

- 長いビデオ（10s+）はタイムラインセグメンテーションを使用：`0-3秒：[説明] / 3-6秒：[説明]`
- アクション/感情は具体的に：❌ `キャラクターが悲しい` → ✅ `涙が頬を流れ、口角が微かに震える`
- ワンショットは最後に追加する必要があります：`全体を通してカットしないでください、ワンショット。`

**キーワードが特定の効果をトリガー**

| 望ましい効果 | 推奨される書き方 |
|------------|----------|
| ヒッチコック・ズーム | `主人公が恐怖を感じるときヒッチコック・ズーム` |
| 円形カメラ | `ロボットアーム多角度円形` |
| 速度が徐々に速くなる | `ジェットコースターの速度がどんどん速くなる` |
| パーティクル効果 | `金色の砂が散る` / `パーティクル吹き飛ばし効果` |

**マルチモーダル組み合わせ戦略**

| 制御したい次元 | 使用する素材 |
|----------------|------------|
| キャラクター外観 | 画像（複数角度） |
| カメラ移動方法 | ビデオ（カメラ言語参照） |
| アクション | ビデオ（アクション参照） |
| 音色/トーン | ビデオ（対話を含む参照ビデオ） |
| 背景音楽リズム | ビデオまたはオーディオ |
| シーンスタイル | 画像（シーン参照画像） |

**よくある質問**

- **参照ビデオがカメラ移動を複製していない？** → `@ビデオ1 のすべてのカメラ移動効果を完全に参照` を追加
- **キャラクターの外観が一貫していない？** → 複数角度の画像をアップロード、プロンプトに `キャラクター外観を @画像1 と完全に一致させる` を追加
- **ビデオ延長の継ぎ目が不自然？** → 延長プロンプトの冒頭で元のビデオの最後のフレームの状態を説明

## 📝 プロンプトテンプレート

**製品 360 度表示**
```
@画像1 の[製品名]をメインとして、カメラ移動は @ビデオ1 を参照、[クローズアップ部分]にズームイン、
カメラ回転後[製品]が全体を反転表示、[製品の特色詳細]が明確に見える、
周囲の環境[雰囲気説明]
```

**広告比較**
```
これは[製品]広告です、@画像1 を最初のフレーム画面として、[キャラクターA]が[状態A、例：優雅]、
カメラが右側に素早くパン、@画像2 [キャラクターB][状態B、例：みすぼらしい]を撮影、
カメラが左側にパンして[製品]を撮影、[製品]は @画像3 を参照、[製品]が[動作状態]。
```

**ビデオ延長スクリプト**
```
[N]秒
@ビデオ1 を[前方/後方]に [N] 秒延長。
[0-X]秒：[画面説明]。
[X-Y]秒：[画面説明]。
[Y-N]秒：[終了画面/字幕]。
```

**ワンショット**
```
@画像1@画像2@画像3...、[視点]ワンショット[移動方法]ショット、
[移動軌跡：AからBを経由してCに到達]、[速度/リズム変化]。
全体を通してカットしないでください、ワンショット。
```
