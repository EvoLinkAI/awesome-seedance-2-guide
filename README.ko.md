## ✨ Seedance 2.0은 왜일까?

**이미지 + 비디오 + 오디오 + 텍스트** 4가지 모달리티를 동시 입력으로 지원합니다. `@소재명`을 자연어로 사용하여 원하는 효과를 설명하면 모델이 이해합니다. 단순한 "생성"이 아니라 진정한 제어 가능한 창작입니다.

## 📋 매개변수 사양

| 입력 유형 | 지원 형식 | 수량 제한 | 크기 제한 | 기간 제한 |
|----------|----------|----------|----------|----------|
| 이미지 | jpeg, png, webp, bmp, tiff, gif | ≤ 9 | 단일 < 30MB | — |
| 비디오 | mp4, mov | ≤ 3 | 단일 < 50MB | 총 2-15s |
| 오디오 | mp3, wav | ≤ 3 | 단일 < 15MB | 총 ≤ 15s |
| 텍스트 | 자연어 | — | — | — |

**결합 제한:** 총 ≤ 12개 파일 (이미지 + 비디오 + 오디오 합계)

**출력 사양:** 생성 기간 4-15s 자유롭게 선택 가능, 비디오 최대 720p, 음향 효과/배경음악 포함

> **준수 사항:** 현재 현실적인 인간 얼굴을 포함하는 자료 업로드를 지원하지 않습니다. 일러스트레이션 스타일, AI 생성 가상 캐릭터, 동물, 제품, 장면 등의 사용을 권장합니다.

## 🎮 상호작용 방법

프롬프트에서 `@소재명`을 사용하여 각 소재의 역할을 지정합니다. 업로드 순서는 번호 매기기 순서입니다:

```
@이미지1 을 첫 번째 프레임으로, @비디오1 을 카메라 언어 참조로, @오디오1 을 배경음악으로
```

| 진입점 | 사용 사례 |
|------|----------|
| **첫 번째/마지막 프레임** | 첫 번째 프레임만 (또는 첫 번째 + 마지막 프레임) + 프롬프트 업로드 |
| **올인원 참조** | 멀티모달 조합 입력 (이미지 + 비디오 + 오디오 + 텍스트) |

**일반적인 작성 패턴:**

```
# 첫 번째 프레임 지정
@이미지1 을 장면의 첫 번째 프레임으로 사용, ...

# 카메라 이동만 참조, 캐릭터는 참조하지 않음
@비디오1 의 모든 카메라 이동 효과를 참조, 하지만 캐릭터 외모는 @이미지1 사용

# 동작과 카메라 이동을 분리하여 참조
@비디오1 에서 캐릭터 동작을 참조, @비디오2 에서 원형 카메라 이동을 참조

# 비디오 연장 (생성 기간 = 새로운 초, 총 기간 아님)
@비디오1 을 5초 연장, [콘텐츠 설명]

# 비디오 음향 효과 참조
배경 BGM은 @비디오1 의 음향 효과를 참조
```

## 💡 고급 기법

**프롬프트 작성**

- 긴 비디오 (10s+)는 타임라인 세분화 사용: `0-3초: [설명] / 3-6초: [설명]`
- 동작/감정은 구체적으로: ❌ `캐릭터가 슬프다` → ✅ `눈물이 뺨을 따라 흐르고, 입가가 미세하게 떨린다`
- 원샷은 끝에 추가해야 함: `전체를 통해 컷하지 마세요, 원샷.`

**키워드가 특정 효과 트리거**

| 원하는 효과 | 권장 작성법 |
|------------|----------|
| 히치콕 줌 | `주인공이 공포를 느낄 때 히치콕 줌` |
| 원형 카메라 | `로봇 팔 다각도 원형` |
| 속도 점진적 증가 | `롤러코스터 속도가 점점 빨라진다` |
| 파티클 효과 | `금색 모래가 흩어진다` / `파티클 분산 효과` |

**멀티모달 조합 전략**

| 제어하려는 차원 | 사용할 소재 |
|----------------|------------|
| 캐릭터 외모 | 이미지 (다각도) |
| 카메라 이동 방식 | 비디오 (카메라 언어 참조) |
| 동작 | 비디오 (동작 참조) |
| 음색/톤 | 비디오 (대화 포함 참조 비디오) |
| 배경음악 리듬 | 비디오 또는 오디오 |
| 장면 스타일 | 이미지 (장면 참조 이미지) |

**자주 묻는 질문**

- **참조 비디오가 카메라 이동을 복제하지 않음?** → `@비디오1 의 모든 카메라 이동 효과를 완전히 참조` 추가
- **캐릭터 외모가 일관성이 없음?** → 다각도 이미지 업로드, 프롬프트에 `캐릭터 외모를 @이미지1 과 완전히 일치` 추가
- **비디오 연장 이음새가 부자연스러움?** → 연장 프롬프트 시작 부분에서 원본 비디오의 마지막 프레임 상태 설명

## 📝 프롬프트 템플릿

**제품 360도 표시**
```
@이미지1 의 [제품명]을 주체로, 카메라 이동은 @비디오1 참조, [클로즈업 부분]으로 줌인,
카메라 회전 후 [제품]이 전체를 반전 표시, [제품 특색 세부사항]이 명확히 보임,
주변 환경 [분위기 설명]
```

**광고 비교**
```
이것은 [제품] 광고입니다, @이미지1 을 첫 번째 프레임 화면으로, [캐릭터A]가 [상태A, 예: 우아함],
카메라가 오른쪽으로 빠르게 팬, @이미지2 [캐릭터B][상태B, 예: 초라함]을 촬영,
카메라가 왼쪽으로 팬하여 [제품]을 촬영, [제품]은 @이미지3 참조, [제품]이 [작동 상태].
```

**비디오 연장 스크립트**
```
[N]초
@비디오1 을 [앞으로/뒤로] [N]초 연장.
[0-X]초: [화면 설명].
[X-Y]초: [화면 설명].
[Y-N]초: [종료 화면/자막].
```

**원샷**
```
@이미지1@이미지2@이미지3..., [시점]원샷 [이동 방식]샷,
[이동 궤적: A에서 B를 거쳐 C에 도달], [속도/리듬 변화].
전체를 통해 컷하지 마세요, 원샷.
```
