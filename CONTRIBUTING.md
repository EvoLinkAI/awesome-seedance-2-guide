# 贡献指南

感谢您对 即梦 Seedance 2.0 使用手册 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 1. 报告问题

如果您发现文档中的错误或需要补充的内容：

1. 点击 GitHub 仓库的 **Issues** 标签
2. 点击 **New Issue**
3. 选择适当的模板（Bug 报告或功能建议）
4. 详细描述问题或建议

### 2. 提交改进

#### 小型修改（错别字、格式调整）

1. 在 GitHub 上直接编辑文件
2. 提交 Pull Request

#### 大型修改（新增章节、案例）

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开一个 Pull Request

### 3. 分享案例

如果您有优秀的使用案例想要分享：

1. 按照 `examples/case-xx-name.md` 的格式创建文件
2. 包含以下内容：
   - 场景描述
   - 使用的素材（图片描述即可）
   - Prompt 文本
   - 效果说明
3. 提交 Pull Request

---

## 内容规范

### 文档格式

- 使用 Markdown 格式
- 文件编码：UTF-8
- 使用中文标点符号

### 图片规范

- 优先使用文字描述，避免上传图片
- 如需引用官方图片，请注明来源
- 不要上传包含写实人脸的图片

### Prompt 规范

```markdown
### 案例名称

**场景描述：**
简要描述这个案例的使用场景

**输入素材：**
- 图片1：描述
- 视频1：描述
- 音频1：描述

**Prompt：**
```
@图片1 作为首帧，参考@视频1的运镜...
```

**效果说明：**
描述预期效果和注意事项
```

## 审核流程

1. 提交 Pull Request
2. 维护者进行审核
3. 如有问题，会提出修改建议
4. 审核通过后合并到主分支

- 接受建设性批评
- 关注对社区最有利的事情
- 尊重不同的观点和经验
